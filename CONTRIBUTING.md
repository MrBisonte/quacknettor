# Contributing to DuckEL

Thank you for your interest in contributing to DuckEL! We welcome contributions from everyone.

## Getting Started

1.  **Fork the repository** on GitHub.
2.  **Clone your fork** locally:
    ```bash
    git clone https://github.com/YOUR_USERNAME/quacknettor.git
    cd quacknettor
    ```
3.  **Install dependencies**:
    ```bash
    pip install .
    ```

## Running Tests

We use `pytest` for testing. Ensure you have it installed:
```bash
pip install pytest
```

Run the tests:
```bash
pytest
```

## Submitting Changes

1.  Create a new branch for your feature or fix.
2.  Make your changes.
3.  Run tests to ensure nothing is broken.
4.  Commit your changes with descriptive messages.
5.  Push your branch to your fork.
6.  Open a **Pull Request** against the `main` branch.

## Code Style

-   Follow PEP 8 guidelines.
-   Use type hints for function signatures.
-   Add docstrings to new functions and classes.
